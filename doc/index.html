Source Code

Requirement
The source code must be stored in a permanent and accessible location.
Justification
… to ensure the source code is not lost.
Requirement
The source code must be stored in a Git repository.
Justification
… to be compatible with Heroku.
Solution
Use GitHub.
Justification
GitHub is free and best-of-breed.

Registrar & DNS

Requirement
The domain name must be registered.
Requirement
The DNS records must be manually editable.
Justification
… to route e-mail to Google Apps, to route web requests to Heroku, and to configure SPF to prevent spam.
Solution
Use Namecheap.
Justification
Namecheap is cheap and easy to use.

E-mail

Requirement
E-mail services.
Solution
Use Google Apps.
Justification
Google Apps is free, familiar, and convenient.
Requirement
It must be possible to send over 1000 e-mails in a day.
Justification
… to send sellers reclaim reminders, in particular.
Solution
Use SendGrid.
Justification
There's no competitor with better reliability and lower cost.

Deployment

Requirement
The application must be easy to deploy.
Requirement
The application must run on a Ruby on Rails stack.
Requirement
The application must have sufficient memory available to it.
Solution
Use Heroku.
Justification
Deploying an application and configuring and maintaining a Ruby on Rails stack require programming skills; thus, a managed solution is the best long-term strategy. Heroku makes deployment very easy, manages the system architecture, and makes sufficient memory available.
